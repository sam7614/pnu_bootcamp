# 부산대학교 부트캠프 훈련생 관리 웹페이지 개발 계획서

## 1. 개발 개요

본 웹페이지는 부산대학교 부트캠프에 참여하는 학생들의 교과형 교육과정, 몰입형 교육과정, 상담 이력, 트랙별 이수 현황을 통합 관리하기 위한 관리자 중심 웹앱이다. 교수자와 운영자가 학생별 이수 상태를 빠르게 파악하고, 다음 교육과정 또는 PBL 과정을 상담·추천할 수 있도록 대시보드와 학생관리 화면을 제공한다.

현재 웹앱은 부트캠프 트랙별 이수체계도를 기준으로 학생 데이터를 모듈 단위로 관리하며, 연도·학기·과정별 필터링, 트랙별 현황, 상담 우선순위, 학생별 추천 과정 정보를 한 화면에서 확인할 수 있도록 설계한다.

## 2. 개발 목적

1. 학생별 교육과정 이수 현황을 체계적으로 관리한다.
2. 교수자가 상담 전 학생의 학년, 전공, 관심 분야, 이수 교과목, 미이수 교과목을 빠르게 확인할 수 있도록 한다.
3. 학생별 이수 데이터를 기반으로 적절한 중급·고급 교과형 과정과 몰입형 PBL 과정을 추천한다.
4. 부트캠프 운영자가 전체 학생의 참여 현황, 트랙별 편중, 이수율, 상담 필요 대상을 대시보드에서 즉시 파악할 수 있도록 한다.
5. 상담 음성파일 업로드와 상담 요약 기능을 통해 교수자의 상담 기록 작성 부담을 줄인다.

## 3. 개발 목표

### 3.1 관리자 대시보드 제공

관리자는 첫 화면에서 현재 진행 중인 트랙별 현황, 관리 학생 수, 평균 이수율, 상담 우선 대상, PBL 추천 가능 학생 수를 확인할 수 있다. 대시보드는 연도별·학기별·과정별 조건을 적용해 학생 현황을 집계한다.

### 3.2 중급과정·고급과정 분리 관리

중급과정과 고급과정을 별도 대시보드로 구분한다. 중급과정은 코어AI트랙 중심으로 구성하고, 고급과정은 트랙별 교과형과 몰입형 PBL을 나누어 관리한다.

- 중급과정: 코어AI트랙
- 코어AI 세부 모듈: 클라우드·피지컬AI, 생성형AI, XR공간컴퓨팅, AI융합보안
- 융합AI: 중급 교과형이 아닌 몰입형 과정으로 관리
- 융합AI 몰입형: 딥러닝 PBL, 데이터분석 PBL
- 고급과정 교과형: 클라우드 AI, 피지컬 AI, 생성형 AI, XR공간컴퓨팅, AI융합보안, 산업 AI
- 고급과정 몰입형: 클라우드 AI PBL, 온디바이스 AI PBL, 로보틱스 AI PBL, 생성형 AI PBL, XR PBL, AI융합보안 PBL, 산업AI PBL

### 3.3 학생별 이수 이력 관리

학생별로 이수한 교과목과 이수해야 할 교과목을 구분해 보여준다. 이수 내역은 연도와 학기로 구분하여 관리하며, 동일 학생의 학기별 성장 흐름을 확인할 수 있도록 한다.

### 3.4 추천 기능 제공

학생의 전공, 학년, 목표 트랙, 관심 분야, 이수 교과목, 미이수 교과목을 바탕으로 추천 가능한 트랙과 추천 이유를 제공한다. 추천 정보는 별도 탭이 아니라 학생 이름을 클릭했을 때 하단에 펼쳐져 교수자가 상담 중 바로 참고할 수 있도록 한다.

### 3.5 상담 이력 관리

학생 상담 내용은 관리자 페이지에서 기록하고, 향후 음성파일 업로드 시 상담 내용을 자동 요약해 상담 이력에 저장할 수 있도록 한다. 음성상담 기능은 실제 운영 단계에서 음성 인식과 요약 API를 연동해 확장한다.

## 4. 주요 사용자

### 4.1 교수자

학생의 이수 현황과 추천 과정을 확인하고, 상담 시 다음 학기 또는 다음 과정 이수 계획을 안내한다.

### 4.2 부트캠프 운영 관리자

전체 학생 데이터를 관리하고, 트랙별 목표 학생 현황, 이수율, 상담 우선순위, PBL 추천 대상을 모니터링한다.

### 4.3 교육과정 기획자

트랙별 학생 분포와 이수율을 확인하여 교육과정 개선, 과목 개설, PBL 운영 계획 수립에 활용한다.

## 5. 핵심 기능

### 5.1 대시보드

- 관리 학생 수, 평균 이수율, 상담 우선 대상, PBL 추천 가능 학생 수 표시
- 각 항목 클릭 시 하단에 관련 학생 리스트와 요약 현황 표시
- 학생 리스트는 10명 단위 페이지네이션 제공
- 연도별·학기별·과정별 필터 제공
- 트랙별 목표 학생 현황과 이수율 표시
- 상담 우선순위 학생 목록 표시
- 현재 진행 중인 중급과정·고급과정 현황 표시

### 5.2 트랙 및 과정 현황

- 중급과정과 고급과정을 별도 영역으로 구분
- 고급과정 대시보드는 한 줄에 3개 트랙씩 2줄로 정렬
- 교과형과 몰입형을 구분해 이수율 표시
- 트랙 또는 과정 유형 클릭 시 팝업으로 세부 이수 현황 표시
- 팝업 내 학생 리스트, 이수 완료 과목, 이수 필요 과목 표시

### 5.3 학생관리

- 100명의 샘플 학생 데이터 기반 관리
- 이름, 전공, 학년, 관심 분야, 목표 트랙, 상담 상태, 위험도 관리
- 학생 선택 시 상담 카드와 이수 현황 표시
- 상담 메모 작성 및 상담완료 처리
- CSV 내보내기 기능 제공

### 5.4 수강이력 관리

- 학생별 모듈 단위 수강 이력 관리
- 교과형·몰입형 구분
- 초급·중급·고급 및 중급형·고급형 단계 구분
- 연도별·학기별 이수 내역 확인
- 이수 완료 과목과 미이수 과목 분리 표시

### 5.5 추천 기능

- 학생별 추천 트랙명 표시
- 추천 이유 제공
- 트랙 설명 제공
- 교수자 상담에 필요한 근거 정보 제공
- 학년, 전공, 이수 교과목, 관심 분야, PBL 적합도를 고려한 추천 로직 적용

### 5.6 상담 음성파일 업로드

- 상담 음성파일과 전사문 업로드 영역 제공
- 업로드 후 상담 요약 생성
- 학생 상담이력에 자동 추가되는 구조 제공
- 운영 단계에서는 음성 인식, 텍스트 요약, 개인정보 보호 절차를 포함해 고도화

## 6. 사용성 편의를 위한 설계 방법

### 6.1 첫 화면에서 핵심 현황 확인

관리자가 웹페이지에 접속하면 먼저 현재 진행 중인 트랙별 현황과 주요 KPI를 확인할 수 있도록 한다. 별도의 검색이나 메뉴 이동 없이 전체 흐름을 파악할 수 있어 운영자의 판단 시간을 줄인다.

### 6.2 클릭 기반 상세 정보 제공

대시보드의 카드, 트랙, 과정 유형, 학생 이름을 클릭하면 관련 정보가 아래로 펼쳐지거나 팝업으로 표시된다. 화면 이동을 최소화하여 상담 중에도 필요한 정보를 빠르게 확인할 수 있다.

### 6.3 정보 과밀 방지

학생 목록은 10명 단위로 나누어 보여주고, 상세 정보는 사용자가 선택했을 때만 표시한다. 이를 통해 한 화면에 너무 많은 정보가 몰리지 않도록 한다.

### 6.4 교과형과 몰입형의 명확한 구분

교과형 이수 현황과 몰입형 PBL 이수 현황을 별도로 표시한다. 학생이 어떤 과정을 이미 이수했고, 어떤 과정을 추가로 이수해야 하는지 명확히 구분할 수 있다.

### 6.5 시각적 인지성 강화

트랙별 색상, 배지, 진행률 바, 상태 표시를 활용해 사용자가 텍스트를 모두 읽지 않아도 학생 상태를 빠르게 파악할 수 있도록 한다.

### 6.6 반응형 화면 구성

카드와 학생 행이 화면 너비를 벗어나지 않도록 반응형 그리드와 줄바꿈 규칙을 적용한다. 데스크톱 화면에서는 정보 밀도를 유지하고, 작은 화면에서는 세로 배치로 전환해 가독성을 확보한다.

### 6.7 상담 중심 정보 제공

추천 트랙, 추천 이유, 미이수 과목, 다음 상담일, 위험도, 상담 상태를 학생 단위로 연결해 보여준다. 교수자는 상담 전후에 같은 화면에서 확인과 기록을 할 수 있다.

## 7. 데이터 관리 계획

### 7.1 학생 데이터

학생 데이터는 이름, 전공, 학년, 목표 트랙, 관심 분야, 상담 상태, 위험도, 상담 메모, 다음 상담일, 이수 모듈 목록으로 구성한다.

### 7.2 교과목 및 모듈 데이터

교과목 데이터는 모듈명, 과정 유형, 단계, 권장 학기, 연결 트랙, 키워드로 구성한다. 이를 통해 트랙별 이수율과 추천 로직을 계산한다.

### 7.3 상담 데이터

상담 데이터는 상담 일자, 상담 요약, 상담 파일명, 후속 조치, 추천 과정으로 관리한다. 운영 단계에서는 상담 음성 원본과 전사문 저장 정책을 별도로 수립한다.

## 8. 기술 구현 계획

### 8.1 현재 프로토타입

- HTML, CSS, JavaScript 기반 정적 웹앱
- 브라우저에서 즉시 실행 가능
- 샘플 학생 데이터 100명 포함
- LocalStorage 기반 데모 데이터 관리

### 8.2 운영 시스템 확장

- 프론트엔드: React 또는 Vue 기반 컴포넌트화
- 백엔드: Node.js, Python FastAPI 또는 Spring Boot 기반 API 서버
- 데이터베이스: PostgreSQL 또는 MySQL
- 파일 저장소: 상담 음성파일 저장을 위한 별도 스토리지
- 인증·권한: 관리자, 교수자, 운영자 권한 구분
- 음성상담: 음성 인식 API와 요약 API 연동

## 9. 개발 단계

### 1단계: 화면 프로토타입 구축

대시보드, 학생관리, 수강이력, 트랙맵, 추천 화면을 구현하고 샘플 데이터로 기능 흐름을 검증한다.

### 2단계: 데이터 구조 정교화

실제 부트캠프 교육과정 기준으로 트랙, 모듈, 교과목, PBL 과정, 학생 이수 데이터를 정리한다.

### 3단계: 관리자 기능 고도화

학생 검색, 필터, CSV 내보내기, 상담 상태 변경, 상담 메모 관리, 연도·학기별 이력 조회 기능을 보완한다.

### 4단계: 추천 로직 개선

학생 전공, 학년, 이수 교과목, 관심 분야, 미이수 과목, 트랙 목표를 반영해 추천 정확도를 높인다.

### 5단계: 상담 음성파일 기능 연동

음성파일 업로드, 자동 전사, 상담 요약, 상담 이력 자동 저장 기능을 구현한다.

### 6단계: 운영 배포

사용자 권한, 개인정보 보호, 데이터 백업, 접속 로그, 운영자 교육을 포함해 실제 운영 환경에 배포한다.

## 10. 기대 효과

1. 교수자의 학생 상담 준비 시간이 단축된다.
2. 학생별 이수 현황과 다음 추천 과정이 명확해진다.
3. 운영자는 트랙별 편중과 이수율을 빠르게 확인할 수 있다.
4. 상담 기록과 수강이력이 연결되어 학생 추적 관리가 쉬워진다.
5. 부트캠프 교육과정 운영 데이터를 기반으로 과정 개선과 PBL 운영 계획 수립이 가능해진다.

## 11. 개인정보 및 보안 고려사항

학생 이름, 전공, 상담 기록, 음성파일은 개인정보에 해당하므로 접근 권한을 제한해야 한다. 운영 시스템에서는 로그인 인증, 역할별 권한, 상담파일 암호화 저장, 파일 보관 기간 설정, 삭제 요청 처리, 접속 이력 관리가 필요하다.

상담 음성파일을 외부 인공지능 API로 전송하는 경우에는 학생 동의, 전송 데이터 범위, 보관 여부, 익명화 정책을 명확히 해야 한다.

## 12. 최종 개발 방향

본 웹앱은 단순한 학생 목록 관리 도구가 아니라, 부트캠프 교육과정의 이수 흐름과 교수자 상담을 연결하는 교육 운영 관리 시스템을 목표로 한다. 핵심 방향은 학생 데이터를 모으는 것에 그치지 않고, 이수 현황을 해석하고, 다음 과정 선택을 돕고, 상담 기록을 축적하여 학생별 맞춤형 교육 안내가 가능하도록 하는 것이다.

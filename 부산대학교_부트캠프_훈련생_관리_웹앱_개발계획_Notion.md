# 부산대학교 부트캠프 훈련생 관리 웹앱 개발계획서

> **핵심 요약**
>
> 본 웹앱은 부산대학교 부트캠프 참여 학생의 이수 현황, 상담 이력, 트랙 추천, PBL 추천 정보를 하나의 관리자 화면에서 관리하기 위한 교육 운영 시스템이다. 교수자는 학생별 이수 근거를 바탕으로 다음 교육과정을 상담할 수 있고, 운영자는 트랙별 현황과 상담 우선순위를 대시보드에서 빠르게 확인할 수 있다.

---

## 1. 개발 목적

| 구분 | 목적 |
|---|---|
| 이수 관리 | 학생별 교과형·몰입형 교육과정 이수 현황을 모듈 단위로 관리 |
| 상담 지원 | 교수자가 상담 전 학생의 학년, 전공, 관심 분야, 이수·미이수 교과목을 빠르게 확인 |
| 과정 추천 | 학생 데이터 기반으로 중급·고급 교과형 과정과 몰입형 PBL 과정 추천 |
| 운영 관리 | 전체 학생의 참여 현황, 트랙별 편중, 이수율, 상담 필요 대상을 대시보드에서 확인 |
| 기록 자동화 | 상담 음성파일 업로드 및 요약 기능을 통해 상담 기록 작성 부담 완화 |

## 2. 개발 목표

### 2.1 관리자 대시보드 제공

- 관리 학생 수, 평균 이수율, 상담 우선 대상, PBL 추천 가능 학생 수를 첫 화면에서 확인
- 연도별·학기별·과정별 필터를 통해 조건별 학생 현황 집계
- KPI 카드 클릭 시 관련 학생 리스트와 요약 현황을 하단에 표시

### 2.2 중급과정·고급과정 분리 관리

| 과정 | 관리 방식 |
|---|---|
| 중급과정 | 코어AI트랙 중심으로 관리 |
| 고급과정 | 트랙별 교과형과 몰입형 PBL을 분리해 관리 |
| 융합AI | 중급 교과형이 아닌 몰입형 과정으로 관리 |

### 2.3 학생별 이수 이력 관리

- 학생별 이수한 교과목과 이수해야 할 교과목을 구분
- 연도와 학기 기준으로 이수 내역 확인
- 학기별 성장 흐름과 다음 과정 안내 근거 확보

### 2.4 추천 기능 제공

- 학생 이름 클릭 시 추천 트랙명, 추천 이유, 트랙 설명 표시
- 전공, 학년, 관심 분야, 이수 교과목, 미이수 교과목을 고려
- 교수자가 상담 중 바로 활용할 수 있는 문장형 근거 제공

### 2.5 상담 이력 관리

- 상담 메모, 상담 상태, 다음 상담일 관리
- 상담 음성파일 업로드 영역 제공
- 향후 음성 인식 및 요약 API와 연동해 상담이력 자동 정리

## 5. 핵심 기능

### 5.1 대시보드

- 관리 학생 수 표시
- 평균 이수율 표시
- 상담 우선 대상 표시
- PBL 추천 가능 학생 수 표시
- 트랙별 목표 학생 현황 표시
- 현재 진행 중인 중급·고급과정 현황 표시
- 학생 목록 10명 단위 페이지네이션

### 5.2 트랙 및 과정 현황

- 중급과정과 고급과정 별도 대시보드 제공
- 고급과정 대시보드는 한 줄에 3개 트랙씩 2줄로 정렬
- 교과형과 몰입형을 구분해 이수율 표시
- 트랙 클릭 시 세부 트랙별 이수 현황 팝업 제공
- 교과형·몰입형 클릭 시 관련 학생 현황 및 이수 과목 표시

### 5.3 학생관리

- 100명의 샘플 학생 데이터 기반 관리
- 이름, 전공, 학년, 관심 분야, 목표 트랙, 상담 상태, 위험도 관리
- 학생 선택 시 상담 카드와 추천 근거 표시
- 상담 메모 작성 및 상담완료 처리
- CSV 내보내기 지원

### 5.4 수강이력 관리

- 학생별 모듈 단위 수강 이력 관리
- 교과형·몰입형 구분
- 초급·중급·고급 및 중급형·고급형 단계 구분
- 연도별·학기별 이수 내역 확인

### 5.5 추천 기능

- 추천 가능한 트랙명 표시
- 추천 이유 제공
- 트랙 설명 제공
- 이수한 교과목과 이수해야 할 교과목 분리 표시
- 교수자 상담에 필요한 근거 정보 제공

### 5.6 상담 음성파일 업로드

- 상담 음성파일 업로드
- 전사문 또는 상담 메모 입력
- 상담 요약 자동 생성 구조
- 학생 상담이력에 자동 저장하는 운영 구조

## 6. 사용성 편의를 위한 설계 방법

### 6.1 첫 화면에서 핵심 현황 확인

관리자가 웹페이지에 접속하면 현재 진행 중인 트랙별 현황과 주요 KPI를 바로 볼 수 있도록 구성한다. 메뉴 이동 없이 전체 운영 상태를 판단할 수 있다.

### 6.2 클릭 기반 상세 정보 제공

대시보드 카드, 트랙, 과정 유형, 학생 이름을 클릭하면 필요한 정보가 아래로 펼쳐지거나 팝업으로 표시된다. 상담 중 화면 이동을 최소화한다.

### 6.3 정보 과밀 방지

학생 목록은 10명 단위로 나누어 표시하고, 상세 정보는 선택 시에만 보여준다. 많은 학생 데이터를 관리하면서도 화면이 복잡해지지 않도록 한다.

### 6.4 교과형과 몰입형의 명확한 구분

교과형 이수 현황과 몰입형 PBL 이수 현황을 분리해 보여준다. 학생이 어떤 과정을 이미 이수했고 어떤 과정을 추가로 이수해야 하는지 명확히 확인할 수 있다.

### 6.5 시각적 인지성 강화

트랙별 색상, 진행률 바, 위험도 배지, 상담 상태 배지를 활용하여 사용자가 텍스트를 모두 읽지 않아도 학생 상태를 빠르게 파악할 수 있도록 한다.

### 6.6 반응형 화면 구성

화면 크기에 따라 카드와 학생 행이 재배치되도록 설계한다. 컬럼 밖으로 정보가 벗어나지 않도록 줄바꿈과 그리드 기준을 적용한다.

### 6.7 상담 중심 정보 제공

추천 트랙, 추천 이유, 미이수 과목, 다음 상담일, 위험도, 상담 상태를 학생 단위로 연결해 보여준다. 교수자는 같은 화면에서 확인과 기록을 함께 수행할 수 있다.

## 12. 최종 개발 방향

본 웹앱은 단순한 학생 목록 관리 도구가 아니라, 부트캠프 교육과정의 이수 흐름과 교수자 상담을 연결하는 교육 운영 관리 시스템을 목표로 한다.

최종 방향은 다음과 같다.

- 학생별 이수 데이터를 모듈 단위로 축적
- 교수자 상담에 필요한 추천 근거 제공
- 관리자 대시보드로 전체 운영 상태 파악
- 상담 음성파일과 상담 기록을 학생 이력으로 연결
- 교육과정 개선과 PBL 운영 계획에 활용 가능한 데이터 기반 확보

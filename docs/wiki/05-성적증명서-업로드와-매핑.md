# 05. 성적증명서 업로드와 매핑

## 기능 목적

성적증명서 업로드 기능은 학생이 실제로 이수한 교과목을 부트캠프 트랙 교과목과 연결하기 위한 기능입니다.

교수자나 운영자가 성적증명서를 눈으로 확인하고 과목을 하나씩 대조하는 시간을 줄이는 것이 목적입니다.

## 사용 위치

왼쪽 메뉴 하단에 `성적증명서 업로드` 버튼이 있습니다.

이 버튼을 누르면 PDF 파일을 선택할 수 있습니다.

## 처리 흐름

1. 사용자가 성적증명서 PDF를 업로드합니다.
2. 웹앱이 PDF 안의 텍스트를 읽습니다.
3. 학생 이름, 학번, 전공, 학년 정보를 찾습니다.
4. 성적표에 있는 교과목명을 부트캠프 교과목 목록과 비교합니다.
5. 매핑 가능한 교과목을 후보로 표시합니다.
6. 기존 학생과 일치하면 해당 학생에게 연결합니다.
7. 기존 데이터에 없는 학생이면 신규 학생으로 추가할 수 있습니다.
8. 팝업에서 사람이 직접 확인한 뒤 저장합니다.

## 팝업에서 확인하는 정보

성적증명서를 업로드하면 팝업이 뜹니다.

팝업에는 다음 정보가 표시됩니다.

- 학생 이름
- 학번
- 전공
- 학년
- 기존 학생 매칭 여부
- 성적표에서 찾은 교과목
- 부트캠프 교과목과의 매핑 결과
- 추천 가능 트랙
- 상담자가 확인해야 할 메모

## 버튼의 의미

### 확인

화면에 표시된 학생정보와 교과목 매핑 결과가 맞을 때 누릅니다.

확인을 누르면 해당 결과가 학생 데이터에 저장됩니다.

### 수정

학생 이름, 전공, 학년, 교과목 매핑 결과가 잘못되어 있을 때 누릅니다.

수정 후 다시 확인을 눌러 저장할 수 있습니다.

### 취소

성적표 분석 결과를 저장하지 않을 때 누릅니다.

## 기존 학생과 신규 학생 처리

### 기존 학생

성적표의 이름이나 학번이 기존 학생 데이터와 일치하면 기존 학생에게 이수 내역을 추가합니다.

### 신규 학생

성적표에 나온 학생이 기존 데이터에 없으면 새 학생으로 등록할 수 있습니다.

이 경우 성적표에서 읽은 전공, 학년, 이수 과목을 바탕으로 초기 학생 정보가 만들어집니다.

## 주의사항

현재 기능은 시연용 구조입니다.

실제 운영에서는 다음 사항을 추가로 검토해야 합니다.

- 성적증명서 양식이 달라질 때 인식이 잘 되는지
- 스캔본 PDF도 인식할 수 있는지
- 개인정보가 포함된 PDF를 어디에 저장할지
- 원본 파일을 저장하지 않을지, 저장한다면 얼마나 보관할지
- 학생 동의와 개인정보 처리 절차가 필요한지
- 잘못 매핑된 교과목을 사람이 반드시 검수할 수 있는지

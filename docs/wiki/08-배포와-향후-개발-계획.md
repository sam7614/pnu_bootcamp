# 08. 배포와 향후 개발 계획

## 현재 배포 상태

현재 웹앱은 Vercel에 배포되어 있습니다.

- 배포 URL: https://pnu-bootcamp-student-management.vercel.app
- GitHub 저장소: https://github.com/sam7614/pnu_bootcamp

## GitHub 저장 상태

로컬에서 개발한 웹앱 파일은 GitHub 저장소에 업로드되었습니다.

현재 기본 브랜치에 바로 반영하지 않고, 안전하게 Pull Request로 올려둔 상태입니다.

- PR: https://github.com/sam7614/pnu_bootcamp/pull/1

PR을 병합하면 GitHub 기본 브랜치에도 웹앱 소스가 반영됩니다.

## 현재 포함된 주요 파일

- `index.html`: 웹앱 실행 시작 파일
- `app.js`: 주요 기능과 데이터 로직
- `styles.css`: 화면 디자인
- `assets/curriculum-map.jpg`: 부트캠프 트랙별 이수체계도
- `README.md`: 프로젝트 설명
- `vercel.json`: Vercel 배포 설정
- `pnu_virtual_transcript_*.pdf`: 테스트용 가상 성적증명서
- 개발계획서 Markdown 파일
- 개발계획서 PowerPoint 파일

## 향후 개발 계획 1. 실제 운영 시스템 전환

현재 시스템은 시연용 정적 웹앱입니다. 실제 운영을 위해서는 서버와 데이터베이스가 필요합니다.

필요한 개발 항목은 다음과 같습니다.

- 로그인
- 권한 관리
- 중앙 데이터베이스
- 학생 정보 백업
- 상담 기록 수정 이력
- 파일 업로드 저장소
- 개인정보 보호 정책

관련 이슈:

- [#13 실제 DB·로그인·권한관리 기반 운영 시스템 전환](https://github.com/sam7614/pnu_bootcamp/issues/13)

## 향후 개발 계획 2. AI 자동화 고도화

현재 상담 음성파일과 성적증명서 분석은 시연용 구조입니다.

실제 운영에서는 다음 기능이 필요합니다.

- 음성파일 자동 전사
- 상담 내용 자동 요약
- 스캔본 성적표 OCR
- 교과목명 표기 차이 자동 보정
- AI 분석 결과에 대한 관리자 검수

관련 이슈:

- [#14 실제 AI 음성인식·성적표 OCR API 연동](https://github.com/sam7614/pnu_bootcamp/issues/14)

## 향후 개발 계획 3. 교육과정 운영 분석

학생 데이터가 충분히 쌓이면 다음 분석이 가능해집니다.

- 트랙별 학생 수 변화
- 학기별 이수율 변화
- PBL 참여 전후 성과 분석
- 상담 우선 학생의 이수 개선 여부
- 특정 교과목 이수 후 추천 트랙 변화

## 향후 개발 계획 4. 교수자용 상담 리포트

상담 전 자동으로 학생별 요약 리포트를 생성할 수 있습니다.

예상 리포트 항목은 다음과 같습니다.

- 학생 기본정보
- 목표 트랙
- 이수한 교과목
- 미이수 교과목
- 추천 과정
- 추천 이유
- 상담 메모
- 다음 상담 제안

## 운영 전 최종 확인 목록

실제 학생 데이터를 넣기 전에는 다음 항목을 반드시 확인해야 합니다.

- 개인정보 처리 기준
- 접근 권한 기준
- 파일 보관 기준
- 데이터 백업 방식
- AI API 사용 동의 절차
- 관리자 검수 절차
- 장애 발생 시 복구 절차

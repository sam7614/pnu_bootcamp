# 07. 데이터 관리와 개인정보 주의사항

## 현재 데이터 저장 방식

현재 웹앱은 시연용 정적 웹앱입니다.

학생 데이터, 상담 메모, 성적증명서 분석 결과는 브라우저의 `localStorage`에 저장되는 구조입니다.

이 방식은 빠르게 시제품을 만들고 화면 흐름을 검토하는 데 유용합니다.

## localStorage 방식의 특징

### 장점

- 별도 서버 설치가 필요 없습니다.
- 브라우저에서 바로 실행할 수 있습니다.
- 화면과 기능을 빠르게 검증할 수 있습니다.

### 한계

- 여러 사용자가 같은 데이터를 동시에 공유하기 어렵습니다.
- 다른 컴퓨터에서 접속하면 같은 데이터가 보이지 않을 수 있습니다.
- 브라우저 데이터가 삭제되면 저장 내용도 사라질 수 있습니다.
- 실제 개인정보 운영에는 적합하지 않습니다.

## 개인정보에 해당할 수 있는 정보

이 시스템에서 개인정보 또는 민감한 교육정보로 볼 수 있는 항목은 다음과 같습니다.

- 학생 이름
- 학번
- 전공
- 학년
- 상담 기록
- 위험도 또는 상담 상태
- 성적증명서 PDF
- 이수 교과목
- 추천 과정
- 상담 음성파일

## 실제 운영 시 필요한 보호 조치

실제 학생 데이터를 운영하려면 다음 조치가 필요합니다.

### 로그인과 권한 관리

교수자, 운영 관리자, 시스템 관리자 권한을 구분해야 합니다.

예를 들어 모든 사용자가 모든 학생의 상담 기록을 볼 수 있게 하면 안 될 수 있습니다.

### 데이터베이스 저장

학생 데이터는 개인 PC 브라우저가 아니라 중앙 데이터베이스에 저장해야 합니다.

그래야 여러 사용자가 같은 데이터를 보고, 백업과 복구도 가능합니다.

### 수정 이력 관리

누가 언제 학생 정보를 수정했는지 기록해야 합니다.

상담 기록이나 성적표 매핑 결과는 나중에 확인이 필요할 수 있습니다.

### 파일 보관 정책

성적증명서 PDF와 상담 음성파일은 민감한 자료입니다.

다음 기준을 정해야 합니다.

- 원본 파일을 저장할 것인지
- 저장한다면 어디에 저장할 것인지
- 누가 접근할 수 있는지
- 얼마나 오래 보관할 것인지
- 학생이 삭제를 요청하면 어떻게 처리할 것인지

### AI API 사용 시 주의

음성 인식, 성적표 OCR, 상담 요약을 외부 AI API로 처리할 경우 다음을 확인해야 합니다.

- 외부 서비스로 어떤 데이터가 전송되는지
- 학생 동의가 필요한지
- 전송된 데이터가 학습에 사용되는지
- 결과를 얼마나 신뢰할 수 있는지
- 오류가 발생했을 때 사람이 검수할 수 있는지

## 현재 시연용 데이터의 성격

현재 웹앱의 학생 데이터와 성적증명서 PDF는 기능 검증을 위한 가상 샘플 데이터입니다.

실제 학생 개인정보를 넣어 운영하기 전에는 반드시 보안과 권한 체계를 먼저 마련해야 합니다.

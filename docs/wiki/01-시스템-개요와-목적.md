# 01. 시스템 개요와 목적

## 시스템 개요

부산대학교 부트캠프 훈련생 관리 웹앱은 학생의 교육과정 이수 현황, 상담 이력, 트랙 추천, 성적증명서 기반 교과목 매핑을 통합 관리하기 위한 관리자용 웹앱입니다.

현재 버전은 시연과 기획 검증에 적합한 정적 웹앱입니다. 별도의 서버 설치 없이 브라우저에서 실행되며, 학생 데이터는 브라우저 저장소를 활용합니다.

## 개발 목적

이 시스템의 목적은 교수자와 운영자가 학생 상담을 더 빠르고 정확하게 준비할 수 있도록 돕는 것입니다.

기존 방식에서는 학생별로 다음 정보를 따로 확인해야 했습니다.

- 학생의 전공과 학년
- 학생이 목표로 하는 트랙
- 이미 이수한 교과목
- 앞으로 이수해야 할 교과목
- PBL 참여 가능성
- 상담 메모와 다음 상담 계획

이 웹앱은 위 정보를 한 화면에서 연결해 보여주도록 설계되었습니다.

## 해결하려는 문제

### 1. 학생별 이수 현황 파악의 어려움

부트캠프 교육과정은 트랙, 단계, 교과형, 몰입형으로 나뉘어 있습니다. 학생이 어떤 과정을 얼마나 이수했는지 수작업으로 파악하면 시간이 오래 걸립니다.

### 2. 상담 근거 부족

학생에게 다음 과정을 추천하려면 단순히 관심 분야만 보는 것이 아니라 이수 교과목, 학년, 전공, 목표 트랙을 함께 봐야 합니다. 이 시스템은 추천 이유를 함께 보여주어 교수자가 상담 근거로 활용할 수 있게 합니다.

### 3. 운영 현황의 분산

운영자는 전체 학생 수, 상담 우선 학생 수, 트랙별 이수율, PBL 추천 가능 인원을 빠르게 파악해야 합니다. 대시보드는 이 정보를 첫 화면에서 확인할 수 있게 합니다.

### 4. 성적증명서 확인의 반복 작업

성적증명서에 나온 교과목을 부트캠프 트랙 교과목과 하나씩 대조하는 일은 반복적입니다. PDF 업로드 기능은 이 과정을 자동화하는 방향을 제시합니다.

## 현재 구현된 핵심 기능

- 운영 대시보드
- 학생관리 화면
- 학생별 수강이력 관리
- 트랙맵 확인
- 추천 화면
- 학생별 상담 메모와 상담이력
- 상담 음성파일 업로드 흐름
- 성적증명서 PDF 업로드와 교과목 매핑 확인
- 트랙, 교과목, PBL 안내 팝업
- 전역 검색
- Vercel 배포
- GitHub 저장소 업로드

## 현재 버전의 성격

현재 버전은 실서비스 바로 투입용이라기보다 다음 목적에 적합합니다.

- 교수자와 운영자의 요구사항 확인
- 부트캠프 교육과정 구조를 화면으로 검증
- 학생 상담 화면 흐름 검토
- 성적증명서 업로드와 추천 로직의 가능성 확인
- 향후 실제 데이터베이스 기반 시스템 개발 전 시제품으로 활용

## 향후 운영 시스템으로 확장할 때 필요한 것

실제 운영 시스템으로 쓰려면 다음 기능이 추가되어야 합니다.

- 로그인과 권한 관리
- 중앙 데이터베이스
- 학생 개인정보 보호 정책
- 상담 파일 저장소
- 파일 업로드 보안
- 백업과 복구
- 수정 이력 관리
- 실제 AI 음성 인식 및 OCR API 연동
